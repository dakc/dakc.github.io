About majidai

majidai is a nodejs web framework released under MIT liscense. It works on native node environment without installing any other libraries. And it is less then 50KB in size.
That is why it is very simple and lightweight.

It is a configuration centralized webframework. We can easily create HTTPS SERVER (including http2) by just customizing the configuration.

Installation

Install the framework by using following command. 
npm install majidai

Setup

The structure for writting code will be as following. 
// 1. import majidai
const majidai = require("majidai");

// 2. create instance
const server = new majidai();

// 3. create routing
server.METHOD("ROUTING", function (app) {
    // app object contains different properties and methods
    // which we will be discussing below

    // do codings

    // return response
});

// 4. start listening server
server.start();

※ Change METHOD with either "get" or "post". And ROUTING with the appropriate route.

Let us Create a file named server.js and paste following content. 
// import majidai
const majidai = require("majidai");

// create instance
const server = new majidai();

// create routing
server.get("/", function (app) {
    return "Hello majidai";
});

// start listening server
server.start();
Run server by running following command. 
node server.js

Now, your server will be listening at locahost with 80 port. If the port is not open then, it will throw error.

※ Note

majidai has 3 properties for handling ROUTING.
  1. get
  2. post
  3. customRouting
The "get" property will listen to GET method, while "post" property will listen to POST method. The "customRouting" property will help us to listen for multiple http methods for a single routing.

For example, in the below example we will be able to access "home" path by both GET and POST method. 

// import majidai
const majidai = require("majidai");

// create instance
const server = new majidai();

// create routing
server.customRouting({method:["GET", "POST"], routing:"/home"}, function (app) {
    return "Hello majidai";
});

// start listening server
server.start();

Configuration

The Default Configuration will be as following. 

{
    port: 80,    // port number
    host: "0.0.0.0",    // host ip 
    log: {
        folder: "./log",    // folder where to write log
        isWriteAccess: false,      // if is set to true, it will write access information like time, browser information, url, etc into a log file
    },
    session:
    {
        isActivate: true, // if no data is required to save into session, changing it to false will decrease IO stress.
        timeOut: 1000 * 60 * 5, // miliseconds
    },
    publicDir: "./public",  // the folder to serve static files like css,js,images,etc
    isProduction: true, // if set false it will output all the logs on console
    maxBodySize: 100 * 1024,  // maximum size allowed to body in byte
    header: {    // default header configuration
        "x-content-type-options": "nosniff",
        "x-frame-options": "SAMEORIGIN",
        "x-xss-protection": "1; mode=block",
        "server": "majidai@1.0",
        "Access-Control-Allow-Origin":"*",
    }
}

We can change by passing the Configuration while creating majidai instance.

Let us create a server listening on 8000 port with access log activated and start as development mode.

// import majidai
const majidai = require("majidai");

// create configuration
var config =  {
    port: 8000, // change the port to listen
    isProduction: false, // it will show the logs on console
    log: {
        isWriteAccess: true, // it will write the access log
    }
};

// create instance
const server = new majidai(config);

// create routing
server.get("/", function (app) {
    return "Hello majidai";
});

// start listening server
server.start();

Docker

Let's pull majidai image. 
docker pull dakc/majidai
create container with name majidai and run on background. 
docker run -d -p 8000:80 --name majidai dakc/majidai
Lets watch log. 
docker logs majidai -f

Open Browser and access to access to http://localhost:8000/.
We will be seeing client information in console.

We can open majidai container in interactive mode through following command.

docker exec -it majidai sh

We will be jumped to /data/examples directory where there are samples for various usage. File name will represent the summary of content inside it.

※ OR WE CAN JUST TRY WITH THIS ONE LINE COMMAND.

docker run -it --rm -p 8000:80 --name majidai dakc/majidai sh

Open Browser and access to access to http://localhost:8000/

Access Logging

It will log for each access in json format to a file with name yyyy-mm-dd.access

For removing IO stress, By default it is set to OFF. But we can ON it through configuration if needed.

var config = {
    log: {
        isWriteAccess: true
    }
}

This is not accesible but we can activate and deactivate through isWriteAccess flag.

Session - put

app.session.put - <key>, <value>
majidai will help you to manipulate data easily into session. put method will save the data into session.
parameter type description
key string key to data
value any (number,boolean,string,object) value for key.

Let's try it, 

// put a data in session with key "price" and value 100
app.session.put("price", 100);

// put object
var userObj = {id:1,name:"dakc"};
app.session.put("user",userObj);

Session will be valid for the the time set in configuration. If the time is expired all the datas will be lost automatically.

We can change the session time through configuration. 

// configuration for setting session active time for 5min
var config = {
    session:{
        timeOut: 1000 * 60 * 5
    }
}

Session - get

app.session.get - <key>?
This method will get the data from session.
parameter type description
key string (optional) key to get data

Let's try it, 

// lets get the value of price
var price = app.session.get("price");

// lets get all data from session (do not pass any argument)
var sessionData = app.session.get();

If no argument is passed, it will return all the data from session.

Session - delete

app.session.delete - <key>
This method will delete the data for given key from session.
parameter type description
key string key to remove from session

Let's try it, 

// lets delete the data with key "price" from session
app.session.delete("price");

Session - destroy

app.session.destroy -
This method will delete all the data from session.

Let's try it, 

app.session.destroy();

Session - regenerate session id

app.session.regenId -
This method will regenrate the session ID.

Let's try it, 

app.session.regenId();

※ It will DESTROY all the datas put before the method is called. So, Be carefull while saving data. Saving data after calling this method wont delete.

GET method parameters

app.data.getParams - <key>?
This method will get the data sent as GET parameters.
parameter type description
key string (optional) key to get data

Let's try it, 

// lets get the value of price (../home?price=230)
var price = app.data.getParams("price");

// get all data
var getData = app.data.getParams();
// get value for id (eg, url...?id=yuyuu)
var idSent = getData.id;

If no argument is passed, it will return all the data in JSON format.

key string (optional) key to get data

// get all data send by POST method

If no argument is passed, it will return all the data in JSON format.

Respond Error

app.respond.error - <error code>
This method will respond error.
parameter type description
error code number error code like, 401,500,..

Let's try it, 

// lets respond with internal server error
return app.respond.error(500);

We can customize the error response by using "onError" method.

server.onError(function (errObj) {
    errObj.contentType = "text/html";
    errObj.errMsg = `
<html>
<head>
    <title>custom error</title>
</head>
<body>
    <h1><span style='color:#ff0000'>Woops!</span> Something Went Wrong</h1>
    <p>${errObj.errMsg}</p>
</body>
</html>
`;

    return errObj;
});

In this case all the error response will respond as text/html with the content set above.
See here for usage.

The argument "errObj" is a json object having response code, response message and content-type as data. 

errObj = {
    contentType:　"content-type(text/html,text/plain)",
    errCode: "status code(404,500.,,)",
    errMsg: "response body"
}

We will overwrite the value and return the object.

Authentication - trigger login check

app.triggerAuthCheck -
This method works with mustBeAuthorized method. If user access to a page having mustBeAuthorized before accessing to a page having triggerAuthCheck, then the user will not be able to see the contents below mustBeAuthorized. After user is authenticated call this function otherwise , mustBeAuthorized wont work. This function will regenrate session id automatically.

Let's try it, 

// login page
server.post("/login", function (app) {
    // get post data
    var postParams = app.data.postParams();
    
    // get login id (form input name=login_id)
    var userId = postParams.login_id;
    
    // get login password (form input name=login_pass)
    var userPass = postParams.login_pass"";
    
    // check user id & pass (user authentication)
    var isLogged = 1;
    
    if (isLogged){
        // trigger login check
        app.triggerAuthCheck();

        // remember user id 
        app.session.put("user_id", userId);

        // redirect to /home
        return app.respond.redirect("/home");
    }
    
    // unauthorized login
    return app.respond.redirect("/");
});

Authentication - validate user

app.mustBeAuthorized - <param>?
This method works with triggerAuthCheck method. After user is authenticated call triggerAuthCheck method. And for the pages where you want to give access to only logged user, call mustBeAuthorized method at the very first.
parameter type description
url object object with key url and value to redirect if the user is not logged
msg string response message if the user is not logged
without any parameter it will response with unauthorized access.

Let's try it, 

// authorized user's page
server.get("/membership", app => {
    // redirect if not logged user
    // triggerAuthCheck should be called before otherwise it will always redirect to given url "./"
    if (!app.mustBeAuthorized({url:"./"})) return;
    
    // or send some messages
    if (!app.mustBeAuthorized("some messages as string")) return;

    // or without any parameters (it will return 401 with unauthorized access message)
    if (!app.mustBeAuthorized()) return;

    // do some stuffs

    // return data for logged users only
    return { name: "adfa", dasdf: 323 };
});
